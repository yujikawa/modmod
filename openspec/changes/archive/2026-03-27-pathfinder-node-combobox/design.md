## Context

`PathFinderTab.tsx` の `NodeSelect` コンポーネントはネイティブ `<select>` を使用している。ノード数が多いモデルではドロップダウンをスクロールして探す必要があり、操作性が低い。既存の `groupedNodes`（ドメインでグループ化されたノード一覧）は再利用できる。

現状の `NodeSelect` は3箇所で使われている：
- Single モード: ノード指定（1つ）
- Path モード: From（1つ）、To（1つ）

追加ライブラリは存在しない（ShadCN に Combobox なし、cmdk 未導入）。

## Goals / Non-Goals

**Goals:**
- テキスト入力でノード名・IDを部分一致フィルタできる
- 入力フォーカス時にドロップダウンが展開し、ドメインでグループ化された候補を表示する
- 選択後はテキスト欄にノード名を表示する
- クリアボタンで選択を解除できる
- 既存の `groupedNodes` ロジックをそのまま流用する

**Non-Goals:**
- キーボードナビゲーション（↑↓ キー選択）は今回スコープ外
- 外部ライブラリ（cmdk 等）の導入
- PathFinder 以外のコンポーネントへの適用

## Decisions

### NodeCombobox を PathFinderTab 内のローカルコンポーネントとして実装する

**決定**: `NodeSelect` を `NodeCombobox` に置き換え、PathFinderTab 内にローカル定義する。

**理由**: 現時点で他のコンポーネントからの再利用要求がない。PathFinderTab 専用のドメインコンテキスト（`groupedNodes`）に依存しているため、外部コンポーネント化の恩恵が薄い。ローカルにとどめることでスコープを最小化できる。

**代替案と却下理由:**
- `ui/combobox.tsx` として共通コンポーネント化: 再利用先がないうちは過剰な抽象化

---

### フィルタはクライアントサイドの部分一致（toLowerCase 比較）

**決定**: `query.toLowerCase()` で `n.name` と `n.id` の両方に対して `includes()` 一致。

**理由**: ノード数は通常100以下。サーバーサイドや debounce は不要。

---

### 状態管理は `useState` 2つのみ（query, isOpen）

**決定**: コンボボックス内部の状態として `query`（フィルタ文字列）と `isOpen`（ドロップダウン表示フラグ）のみ管理する。

**理由**: シンプルな UI には十分。`useReducer` や Context は不要。

---

### クリックアウト検知は `useRef` + `mousedown` イベント

**決定**: コンボボックス全体を `ref` で参照し、`document.addEventListener('mousedown', ...)` で外部クリックを検知してドロップダウンを閉じる。

**理由**: `onBlur` は子要素クリック時にも発火するため誤作動しやすい。`mousedown` + `contains()` チェックが最も確実。

---

### 選択済み状態の表示

- `value`（ID）が設定されている場合: 対応するノードの `name` を input に表示、フィルタは無効
- `value` が空の場合: `query` の内容を表示、フィルタ有効
- ユーザーが入力を始めたら `onChange('')` を呼んで選択をリセットし、フィルタモードに戻る

## Risks / Trade-offs

| リスク | 軽減策 |
|--------|--------|
| ドロップダウンが右パネル外にはみ出す可能性 | `max-h` と `overflow-y-auto` でスクロール可能にする。`z-index` を十分に設定する |
| ノード数が非常に多い場合のレンダリング負荷 | `useMemo` でフィルタ結果をメモ化する。実用上は問題ない規模 |
| キーボード操作（↑↓ Enter）が使えない | 今回スコープ外として許容。将来追加可能な設計にしておく |
