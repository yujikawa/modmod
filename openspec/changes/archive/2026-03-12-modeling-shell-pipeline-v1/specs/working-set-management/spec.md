## ADDED Requirements

### Requirement: パイプライン対象のコンテキスト保持
システムは、パイプラインの各工程間で受け渡されるエンティティ ID のリスト（作業セット）を管理しなければならない (SHALL manage working set context)。

#### Scenario: ID リストの継承
- **WHEN** 最初のコマンドが `select fact_*` である場合
- **THEN** 次のコマンドは、このパターンにマッチした ID のリストのみを入力として受け取る

### Requirement: アトミックなパイプライン実行
パイプライン全体の実行は、すべてのステージが正常に完了した後に、一度のファイル保存として行われなければならない (SHALL execute atomically)。

#### Scenario: 複数変更の一括保存
- **WHEN** ユーザーが 3 つのコマンドを繋いだパイプラインを実行し、すべて成功する
- **THEN** ファイルへの `save` リクエストは、すべての変更がストアに反映された後に 1 回だけ行われる
