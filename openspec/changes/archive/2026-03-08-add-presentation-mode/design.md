## Context

開発者向けのIDE風UIから離れ、ビジネスサイドへの説明や合意形成に特化した「プレゼンテーションモード」を導入します。これにより、ノイズを排した全画面表示と、議論の結果を即座に画像共有できる機能を提供します。

## Goals / Non-Goals

**Goals:**
- 左右のパネルと詳細パネルを一括で隠し、キャンバスを最大化する。
- 現在の表示内容を高解像度画像としてエクスポート（スナップショット）する。
- プレゼン中に特定のノードを強調（スポットライト）し、視認性を高める。

**Non-Goals:**
- プレゼン中の高度な編集機能の提供。
- スライドショーのような自動遷移機能（今回は手動操作のみ）。

## Decisions

### 1. モード管理 (`useStore.ts`)
`isPresentationMode: boolean` ステートを追加し、アクティビティバーまたはショートカットでトグル可能にします。

### 2. UIの動的切り替え (`App.tsx`)
`isPresentationMode` が true の場合、以下のコンポーネントを非表示（またはマウント解除）にします。
- `Sidebar` (左)
- `RightPanel` (右)
- `DetailPanel` (下)
代わりに `PresentationOverlay` を表示します。

### 3. スナップショットの実装
`html-to-image` ライブラリを採用し、React Flow の `react-flow__viewport` クラスを持つ要素をキャプチャします。
- **Rationale**: React Flow 標準の `toPng` よりも柔軟なスタイリング制御が可能であり、他のプロジェクトでも実績があるため。

### 4. スポットライト効果
プレゼンモード中、ノードが選択された際に、そのノードの `zIndex` を一時的に上げ、キャンバス全体に `dim-background` クラスを適用（または他のノードの `opacity` を下げる）することで、視線を誘導します。

## Risks / Trade-offs

- **[Risk] 画像書き出しの品質** → ブラウザやテーマによってフォントのレンダリングが崩れる可能性がある。
  - **Mitigation**: 高解像度（pixelRatio: 2）での書き出しをデフォルトとし、主要なブラウザで検証を行います。
- **[Risk] パネル復帰時の不整合** → モードを抜けた際にサイドバーの開閉状態などが崩れる。
  - **Mitigation**: 既存の `isSidebarOpen` などのステートは維持し、プレゼンモード終了時にそのままの状態で復帰するようにします。
