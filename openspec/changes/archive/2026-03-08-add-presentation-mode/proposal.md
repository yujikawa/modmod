## Why

現在のUIは開発効率を重視したIDEライクな構成になっていますが、ビジネスサイドへの説明や合意形成（レビュー）の場面では、サイドバーや詳細パネルがノイズとなり、モデルの本質的な構造への集中を妨げることがあります。また、議論の過程で決定した内容を即座に画像としてエクスポートし、Slackやドキュメントに共有したいというニーズが高まっています。プレゼンテーションモードを導入することで、「見せる・語る・合意する」プロセスを強力にサポートします。

## What Changes

- **プレゼンテーションモード（全画面表示）の導入**: 左右のサイドバーおよび詳細パネルを一括で非表示にし、キャンバスを画面いっぱいに広げるモードを追加します。
- **プレゼン専用オーバーレイUI**: モード有効時に、画面隅に「スナップショット（画像保存）」と「モード終了」のための最小限のボタンを配置します。
- **スナップショット機能**: 現在のキャンバスの状態を高解像度画像（PNG等）としてエクスポート・ダウンロードする機能を実装します。
- **クリック挙動の最適化**: プレゼンテーション中、クリックした要素を強調（スポットライト効果）し、説明対象を明確にする仕組みを導入します。

## Capabilities

### New Capabilities
- `presentation-mode`: プレゼンテーション専用の表示・操作モード。
- `snapshot-export`: キャンバス領域の画像エクスポート機能。

### Modified Capabilities
- `visualizer-core`: モード切り替えに伴うレイアウトの動的変更要件の追加。

## Impact

- `visualizer/src/store/useStore.ts`: `isPresentationMode` ステートの追加。
- `visualizer/src/App.tsx`: モードに応じたコンポーネントの表示・非表示制御。
- `visualizer/src/components/PresentationOverlay.tsx` (新設): プレゼン用ボタン群の実装。
- `html-to-image` または `react-component-export-image` などの画像生成ライブラリの検討（あるいは React Flow 標準ユーティリティの利用）。
