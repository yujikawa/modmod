## Context

Modscape は現在、マウスによるドラッグ＆ドロップでテーブル同士を接続するインターフェースを提供していますが、大規模なプロジェクトではこの操作が負担となります。
また、複数のテーブルに同じ名前のカラム（例：`user_id`）が存在する場合、それらを一つずつ手動で繋ぐのは非常に手間がかかります。

## Goals / Non-Goals

**Goals:**
- キーボード主体でリレーションを作成できるコマンドインターフェースを提供すること。
- テーブル名とカラム名の動的な補完により、タイピング量を最小限に抑えること。
- ワイルドカード（`*`）を使用して、共通カラムを持つ複数のテーブルを一括でソーステーブルに接続できること。

**Non-Goals:**
- 自然言語による高度なリレーション推論（AIによる自動推論は別途検討）。
- 複雑な正規表現によるマッチング（まずは `*` による完全一致または部分一致に留める）。

## Decisions

### 1. `QuickConnectBar` コンポーネント
**内容:** キャンバス上部に `absolute` 配置される、入力フィールドとセレクトボックスからなるコンポーネント。
**トリガー:** キャンバスフォーカス時に `L` キーで表示/非表示をトグル。
**UI:** 
- Source Input (Autocomplete)
- Type Selector (1:N, 1:1, etc.)
- Target Input (Autocomplete + Wildcard support)

### 2. ストアへの `bulkAddRelationship` アクションの追加
**内容:** `sourceTable`, `sourceColumn`, `targetColumnPattern`, `type` を受け取り、現在の `schema` を走査して、パターンにマッチする全テーブルに対してリレーションを作成する。
**重複ガード:** すでに同一のソース・ターゲット間にリレーションが存在する場合は作成をスキップする。

### 3. 自動補完 (Autocomplete) ロジック
**内容:** `useStore` 内の `schema` から、`table_id.column_id` の全リストを生成し、入力文字列に応じてフィルタリングする。
**UX:** `.` を入力した時点で、そのテーブルのカラムリストにサジェストを切り替える。

## Risks / Trade-offs

- **[Risk]** 意図しない一括接続による図面の混乱。
  - **Mitigation:** `bulkAddRelationship` 実行前に、接続対象となるテーブルの数をプレビュー表示（例：「3つのテーブルに接続します」）する仕組みを検討する。
- **[Risk]** ショートカットの衝突。
  - **Mitigation:** `L` キーがエディタ（CodeMirror）内での入力と干渉しないよう、`App.tsx` のグローバルハンドラで `isTyping` 判定を厳密に行う。
