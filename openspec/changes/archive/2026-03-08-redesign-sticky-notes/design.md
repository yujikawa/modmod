## Context

付箋（Annotation）コンポーネントを、より現実の付箋に近い直感的なデザインに刷新します。現在の「Callout」などのバリエーションを廃止し、一つの完成された「紙の質感」を持つデザインに集約します。また、実用上の課題である「長文時の巨大化」と「ライトモードでのゴミ箱UIの視認性」を同時に解決します。

## Goals / Non-Goals

**Goals:**
- 全ての付箋を右下がめくれた（Dog-ear）四角形に統一する。
- 長文が書かれた場合でも、一定の高さで止まり内部スクロールするようにする。
- ライトモードでのゴミ箱（削除）ボタンの配色を全体のテーマに合わせる。
- わずかなランダムな傾きを追加し、デジタルなキャンバスにアナログなアクセントを加える。

**Non-Goals:**
- コネクタ（線）機能の削除（ターゲットがある場合は引き続き線を表示する）。
- 編集ロジック（ダブルクリックでのテキスト編集など）の大幅な変更。

## Decisions

### 1. デザインの統一と Dog-ear の実装
全ての `AnnotationNode` を `borderRadius: 2px` の四角形にします。
- **Dog-ear**: 右下に `16x16px` 程度の `div` を配置し、斜めのグラデーションを用いて「紙の裏側」と「折り目」を表現します。
- **傾き**: ノードの ID をシード値として使用し、`-1.5` から `+1.5` 度の範囲でランダムな `rotate` を適用します。これにより、再描画されても同じ付箋は同じ角度を維持します。

### 2. サイズ制限と内部スクロール
付箋のルート要素またはテキストコンテナに以下の制限を設けます。
- `maxHeight`: `240px`
- `overflowY`: `auto`
- **スタイリング**: 標準の太いスクロールバーを避け、テーマカラーに合わせた細いスクロールバー（`scrollbar-width: thin` または Webkit 擬似要素）を適用します。

### 3. ゴミ箱（削除ボタン）の配色修正
ライトモードにおいて、真っ黒な背景を避けます。
- **Light**: `bg-white`, `border-slate-200`, `text-slate-400`, `hover:bg-red-50`, `hover:text-red-500`
- **Dark**: `bg-slate-800`, `border-slate-700`, `text-slate-500`, `hover:text-red-400`

## Risks / Trade-offs

- **[Risk] テキストの隠蔽** → スクロールが必要なことに気づかないユーザーがいる可能性がある。
  - **Mitigation**: テキストコンテナの下部にうっすらとフェード効果（グラデーション）をかける、またはスクロールバーを常時表示（細く）することで、続きがあることを示唆します。
- **[Risk] 選択枠のズレ** → ノードを回転させると、React Flow の標準の選択枠（Ring）がズレて見える。
  - **Mitigation**: `selected` 時の枠線をインラインスタイルの `boxShadow` や `border` で制御し、回転したノード自体に枠線が追従するようにします。
